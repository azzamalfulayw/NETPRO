PowerShell Global Find & Replace Script

This script is basically a "Regex Find & Replace" execution engine that runs directly against your hard drive. It is extremely powerful for fixing systemic typos or globally renaming variables across thousands of files instantly without breaking binaries or compilation folders.

How it Works:

  • Get-ChildItem -Recurse -File -Filter "*.cs" dynamically searches for every C# file in the source directory.
  • Where-Object explicitly filters out locked/build directories like bin, obj, and Migrations so we don't accidentally corrupt executable builds or Database History.
  • Get-Content -Raw loads the entire file into memory as a singular string.
  • -replace executes a fast string replacement for target words.
  • Set-Content -NoNewline writes the patched string directly back into the hard drive transparently, overriding the old content.

The Script:

$files = Get-ChildItem -Path ".\api" -Recurse -File -Filter "*.cs" | Where-Object { $_.DirectoryName -notmatch "Migrations" -and $_.DirectoryName -notmatch "obj" -and $_.DirectoryName -notmatch "bin" }
foreach ($file in $files) {
    $text = Get-Content $file.FullName -Raw
    $modified = $false
    
    # Add your target words here
    if ($text -match "OldSpelling") {
        $text = $text -replace "OldSpelling","NewSpelling"
        $modified = $true
    }
    
    if ($modified) {
        Set-Content -Path $file.FullName -Value $text -NoNewline
    }
}

How to Run in the Future:

Save the snippet above into a file named replace.ps1 at the root of your project, alter the strings to what you need, and then execute .\replace.ps1 inside a PowerShell terminal instance.

Node.js Global Code Comment Stripper (C#)

A specialized automation script to recursively parse a directory and cleanly remove all single-line (//) and multi-line (/* */) comments from C# source files without accidentally corrupting string literals containing comment-like character sequences (e.g., URL protocol handlers like "https://").

How it Works:

  • fs.readdirSync() traverses the directory tree continuously, isolating active .cs files and willfully avoiding locked artifact folders (bin/obj).
  • Reads the raw contents chunked into JavaScript memory execution.
  • Constructs an advanced Regex tokenization execution: /(".*?"|'.*?'|@".*?")|(\/\*[\s\S]*?\*\/)|(\/\/.*)/g.
  • If a match resolves to a safe string token (capture group 1), the regex gracefully ignores it. If it specifically hits an evaluated comment block (capture group 2 or 3), it mutates it to an empty string.

The Script:

const fs = require('fs');
const path = require('path');

function removeComments(dir) {
    const files = fs.readdirSync(dir);
    for (const file of files) {
        const fullPath = path.join(dir, file);
        if (fs.statSync(fullPath).isDirectory()) {
            // Ignore build output constraints
            if (file === 'bin' || file === 'obj' || file === 'Migrations') continue;
            removeComments(fullPath);
        } else if (file.endsWith('.cs')) {
            let content = fs.readFileSync(fullPath, 'utf8');
            // Advanced regex explicitly preserving C# strings and stripping comments
            content = content.replace(/(".*?"|'.*?'|@".*?")|(\/\*[\s\S]*?\*\/)|(\/\/.*)/g, (match, grp1) => {
                if (grp1) return grp1;
                return '';
            });
            fs.writeFileSync(fullPath, content, 'utf8');
        }
    }
}

removeComments(path.join(__dirname, 'api'));
console.log('Comments successfully stripped.');

How to Run in the Future:

Save this exact script as strip-comments.js at the root structure of your project space. Confirm that a local Node.js environment is available, then boot it by inputting node strip-comments.js inside the system terminal.

Why Gemini API for NETPRO AI Recommendations? (vs. Claude)

A breakdown of the decision matrix behind selecting Google's Gemini API (specifically the Flash models) to power the live Stock AI Recommendations feature, rather than Anthropic's Claude API.

Core Reasoning for Gemini:

  • Generous Free Tier: Gemini offers a highly accessible free tier (e.g., 15 RPM) which is perfect for active, iterative development environments like NETPRO without requiring immediate billing configurations or draining prepaid credits.
  • Lightning Fast Inference (Flash Models): For a UI component expecting synchronous real-time responses ("Live Prices & AI Recommender"), Time-to-First-Token (TTFT) is critical. Gemini Flash excels at rapid output generation, keeping our UI snappy.
  • Reliable JSON Structured Output: Our backend relies on parsing a strict AiRecommendationResponseDto. Gemini's API effortlessly supports schema-enforced JSON outputs out of the box, ensuring the C# backend rarely crashes due to parsing logic.

Comparison against Claude API:

Gemini (Chosen)
  • Ideal for lightweight, high-speed UI endpoints.
  • Easily accessible developer tier for zero-cost prototyping.
  • Highly optimized for REST integrations utilizing explicit JSON boundaries.
Anthropic Claude
  • Arguably superior deep reasoning and stylistic nuance for complex analytical reports.
  • Strictly paid API access (requires pre-funded credits to realistically test).
  • Excellent at following prompt constraints but historically required more manual prompt-engineering for raw JSON extraction.

Conclusion: While Claude is phenomenal for deep systemic analysis, Gemini Flash provided the perfect intersection of zero-friction cost, blistering speed, and easy backend DTO parsing necessary for an interactive web marketplace.
