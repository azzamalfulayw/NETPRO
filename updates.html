📜 Working Rules & Workflow

This section serves as the base reference for all modifications to the NETPRO project.

  • Understand First: Thoroughly inspect and analyze the codebase before suggesting any changes.
  • Explain First: Clearly explain what the project does, its structure, and any identified issues.
  • Propose & Wait: Propose specific changes (what, why, affected files, side effects) and wait for approval before editing anything.
  • No Silent Assumptions: Always discuss unclear points before proceeding.
  • Clean Implementation: Keep code consistent, preserve functionality, and avoid unnecessary refactoring.
  • Document Everything: Every approved change must be logged in this hub.

⚡ AI Recommendation: Lean Analysis Mode

2026-04-07
AI Integration Performance

Simplified the Gemini AI prompt and the resulting AiRecommendationResponseDto to request only a Rating and a concise 1-2 sentence Summary.

Reasoning: Complex structured JSON responses (previously asking for 3-5 specific reasons and warnings) forced the AI to generate excessive tokens. On the heavily loaded free tier, this prolonged generation time resulted in frequent 503 Service Unavailable timeouts.

By shifting to a "Lean Analysis" mode, the generation is lightning fast, substantially increasing the success rate during high-demand periods.

🚀 AI-Powered Stock Recommendations (V1.2 - Error Transparency)

2026-04-07
AI Integration Bug Fix Architecture

Overhauled the application's external API error handling to resolve misleading 404 responses caused by API vendor constraints (Alpha Vantage Rate Limit & Gemini High Demand).

Fixes & Improvements:

  • Strict Error Separation: StockDataService and GeminiRecommendationService now throw explicit HttpRequestException subclasses when encountering external provider blocks, rather than swallowing the failures as null.
  • Transparent HTTP Codes: The AiRecommendationController now accurately returns 429 Too Many Requests for Alpha Vantage rate limits, and 503 Service Unavailable for Gemini model overload. The misleading 404 logic has been isolated exclusively to genuine "Symbol Not Found" cases.
  • Background Throttle: Modified StockPriceUpdateService to wait 10 minutes on startup before executing the background price synchronization. This prevents aggressive `.NET Watch` restarts from consuming the entire Alpha Vantage daily quota during active development.
  • Resilient Grounding Intact: Optional data requests (Company Info, Historicals) are wrapped in specialized try/catches so that rate-limiting on non-essential data still permits a partial recommendation using just the live price.

Technical Code Snip (Controller Catch Block):

catch (System.Net.Http.HttpRequestException hex)
{
    int statusCode = hex.StatusCode.HasValue ? (int)hex.StatusCode.Value : 503;
    string message = statusCode == 429 
        ? "External API rate limit exceeded. Please wait."
        : "The AI service is busy. Please try again later.";
    
    return StatusCode(statusCode, new { error = message, status = statusCode });
}
